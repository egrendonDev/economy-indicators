Status Colors

Green (Healthy) - indicator is within normal historical range. No action needed.
Orange (Watch) - indicator is approaching stress levels. Monitor closely.
Red (Warning) - indicator has crossed a stress threshold. Elevated risk signal.

Healthy Avg

The small gray pill next to each value shows the long-run historical average. Use it as a reference point for what "normal" looks like for that indicator.

Leading vs Lagging

Leading indicators change before the economy does. They give early warning. Watch these first.
Lagging indicators confirm what already happened. Rising delinquencies confirm a downturn is underway.

Cluster-weighted risk score: each of 5 economic clusters (Monetary & Credit 30%, Labor 25%, Valuations 20%, Real Economy 15%, Debt Stress 10%) is scored 0-100% by its own indicators, then combined. This prevents correlated lagging indicators from dominating the score.

Normal Score 0-13%. Economy looks healthy. No meaningful stress signals.

Elevated Score 29-49%. Multiple clusters active. Conditions are tightening.
High Risk Score 50-70%. Broad cluster stress. Downturn likely developing.

Manual Update Guide

These indicators require a file to be manually downloaded and dropped into the dropzone folder. Follow the steps below to update each one.

After updating any JSON file locally, run git add data/ && git commit -m "chore: manual data update" && git push to publish to the dashboard.

FRED-API Updates

These 18 indicators are fetched automatically by GitHub Actions every day at 7am UTC. No manual action needed under normal operation.

To force a local refresh, run npm run api-pull:all (or append -- --force to bypass the freshness check).

HTML-Scraper Updates

These 4 indicators are scraped from multpl.com by running a script manually or via GitHub Actions. The scraper has a 1-hour cooldown guard to avoid hammering the site.

Run npm run html-scrape:multpl to refresh all 4 indicators at once.
